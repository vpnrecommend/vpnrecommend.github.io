为什么要打造图床？

让笔记远走高飞

试问以下场景：

  1. 我们要 markdown 笔记放到某博客上，直接进行复制即可。但因你的图片存储在本地，博客上的图片因无法找到本地路径而无法显示，因此需要到本地找到图片一个个上传
  2. 我们要把 markdown 笔记共享给朋友，需要同时把图片一并打包发送

是否很麻烦？

这时我们可以使用阿里云OSS + PicGo 打造自己的图床让上面的两个场景爽到飞起：

  1. 直接复制 markdown 笔记放入博客即可
  2. 共享笔记给朋友只需要发送一份 markdown 文件

当然你也可以直接选用市面已有的图床，但本文讲的是提供一种打造个人图床的方案

购买阿里云OSS

我这里使用的是阿里云对象存储OSS，你可以使用腾讯、七牛云对象存储等皆可

链接：对象存储OSS_云存储服务_企业数据管理_存储-阿里云 (aliyun.com)

  • 购买阿里云OSS

  1. 点击产品价格可见具体定价情况

可见访问流量也是要收费的，所以确保账户有少量金额

  1. 点击折扣套餐购买存储资源

个人使用存储资源一般 40G 远远够用了，一年定价大概要 9 元

如果在意收费的问题，可以选用七牛云对象存储，每月可提供 10G 的免费存储空间和 10G 的免费 CDN 回源流量，不过需要你拥有域名和进行备案。

  • 进入控制台创建 Bucket

点击 Bucket 列表创建，或右边 Bucket 模块创建皆可

  • Bucket 参数设置
  1. Bucket 名称：自定义
  2. 地域：一般国内任意地域皆可
  3. 存储类型：标准存储
  4. 读写权限：公共读

其他默认即可，或根据自己需求设置

  • 创建AccessKey

鼠标移动至头像，点击 AccessKey 管理即可进入管理界面

创建 AceessKey 记得要保存下来

使用PicGo

下载地址：Releases · Molunerfinn/PicGo (github.com)

下载后安装打开 Picgo

  • 根据自己使用的图床设置，我这里使用的是阿里云OSS

  • 查看 Server 端口是否为 36677（编辑器端口）

  • 图床设置

  1. 设定 KeyID、KeySecret：即之前保存的 AccessKey 与 AccessKeySecret
  2. 设定存储空间名：即之前创建 Bucket 的名字
  3. 确认存储区域：见下

进入到你阿里云OSS的 Bucket 列表点击 Bucket 的名字进入到概览
